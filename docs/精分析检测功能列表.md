# 精分析检测功能列表

本文档根据三类材料整理：

- 23QA/PDF 的核心思想：基于结构理解句意，而不是基于感觉或只看参考译文。
- 豆包检测聊天：围绕同一句原句，依次考核主干、语法成分、重难词、句子切割、固定搭配，最后评级与反馈。
- 精分析笔记点评与 7 月线下班 PPT：七步翻译法为“抄 - 译 - 查 - 改 - 对 - 记 - 滚”，核心是让学生明确自己到底卡在单词、语法、译文表达还是复习不牢。

## 1. 总目标

检测部分不应只是形式化刷题，而要判断学生是否真的完成了精分析训练：

1. 能不能看到原句后先找主干，而不是直接凭语感翻译。
2. 能不能识别主句、从句、短语、一级修饰、二级修饰及其修饰对象。
3. 能不能把长句切成适合翻译的小结构块。
4. 能不能按结构翻译成完整、通顺、准确的中文句子。
5. 能不能把生词、词性、固定搭配、句型放回原句中掌握。
6. 能不能对照标准答案发现自己到底哪里不会。
7. 能不能通过滚动复习，把不熟的语法和单词真正记牢。

选择题可以保留，但只能作为热身和低阶检查。主要检测必须是原句驱动的短答、标注、切分、翻译和自我修正。

## 2. 七步翻译法对应的系统流程

### 2.1 抄

线下动作：学生把要研读的英语句子抄到本子上，每行之间留空，方便划结构和做笔记。

系统功能：

- 显示纯英文原句，不显示翻译、结构块、词义和解析。
- 支持“抄写确认”或“手动重打一遍原句”模式。
- 对长难句和解题阻碍句打标签，提醒学生这类句子需要精分析。
- 可选提供“只看原文”阅读区，让学生不被答案干扰。

必要数据：

- `original` 原句。
- `source` 年份、题型、Text、段落、句号位置。
- `is_key_sentence` 是否为长难句或解题阻碍句。
- `difficulty` 难度。

### 2.2 译

线下动作：先划分句子结构，再写第一遍翻译。此阶段目标不是完美，而是暴露障碍：到底是单词不懂，还是语法结构不懂。

系统功能：

- 主干填写：主语、谓语、宾语/表语/补语。
- 结构标注：学生选择或输入从句、短语、修饰成分，并说明修饰对象。
- 句子切分：学生用 `/` 或拖拽方式切分原句。
- 第一译输入：要求写完整句，而不是词义堆砌。
- 障碍自评：学生选择“单词障碍 / 语法障碍 / 两者都有 / 基本无障碍”。

必要数据：

- `main_structure` 标准主干。
- `chunks` 结构块及角色。
- `relations` 修饰对象关系。
- `standard_cuts` 标准切分。
- `first_translation_rubric` 第一译评分规则。

### 2.3 查

线下动作：查不认识的单词，排除因词义不懂造成的理解障碍。

系统功能：

- 学生先标记不认识的词。
- 系统再显示本句重点词、词性、句中义项、所在结构块。
- 区分“我本来就会 / 查后会了 / 查后仍不懂”。
- 支持把词加入滚动复习。

必要数据：

- `key_words`：词、词性、句中义项、常见义项、所在 chunk。
- `distractor_meanings`：易混义项。
- `word_notes`：例如词性变化、熟词僻义、考研高频用法。

### 2.4 改

线下动作：查词后，在第一次翻译基础上修改译文。重点是自己改，不是抄参考答案。

系统功能：

- 保留第一译，要求学生提交第二译。
- 对比第一译和第二译，提示是否只是照抄答案。
- 要求学生说明：本次修改解决了哪些词义或结构问题。
- 支持“修改依据”勾选：查词、重新识别主干、重新识别从句、调整语序、补全遗漏。

必要数据：

- `translation_literal` 按结构直译。
- `translation_polished` 可选通顺译文。
- `translation_key_points` 译文必须覆盖的要点。
- `common_translation_errors` 常见误译。

### 2.5 对

线下动作：先对照句子结构，再对照参考译文。不能只看中文大意差不多。

系统功能：

- 分步揭示标准答案：先显示主干，再显示修饰层级，再显示切分，再显示译文。
- 学生逐项自查：主干是否正确、从句是否正确、修饰对象是否正确、翻译是否漏点。
- 系统给出错误标签。
- 学生必须写“我错在哪里 / 下次怎么识别”。

必要数据：

- `answer_key.main_structure`
- `answer_key.grammar_points`
- `answer_key.modifier_levels`
- `answer_key.standard_cuts`
- `answer_key.translation`
- `error_tags` 错误标签库。

### 2.6 记

线下动作：记忆句中的单词、词组、语法、翻译方法、句子意思。

系统功能：

- 自动生成本句“记忆清单”。
- 按词汇、短语、语法点、句型分组。
- 支持学生标记掌握程度。
- 根据错误自动生成强化项，例如“主语从句 what 成分判断”“后置定语 vs 状语”。

必要数据：

- `memory_items` 记忆项。
- `grammar_patterns` 句型与语法点。
- `reinforcement_targets` 强化目标。

### 2.7 滚

线下动作：第二天开始新研读前，复习以往句子。难理解、翻译不好的句子要多滚动。

系统功能：

- 自动安排滚动复习：次日、3 天、7 天、14 天。
- 复习时隐藏答案，只显示原句。
- 复习任务不一定完整七步，可只考薄弱项。
- 记录每次是否能快速识别结构、是否能完整翻译。
- 对反复错的语法点生成专项训练。

必要数据：

- `review_history`
- `next_review_at`
- `mastery_level`
- `weakness_tags`
- `review_mode`：主干复习、切分复习、翻译复习、词汇复习、综合复习。

## 3. 检测功能模块

### 3.1 原句精分析综合检测

这是主检测模式。

流程：

1. 系统抽取一句长难句或学生薄弱句。
2. 只显示原句和来源，不显示分析。
3. 学生依次完成：
   - 主干识别。
   - 语法成分与修饰对象。
   - 句子切分。
   - 重点词与固定搭配。
   - 结构直译。
   - 自我判断障碍。
4. 全部提交后再显示标准答案。
5. 系统评分并给 A/B/C/D。
6. 系统输出薄弱点和下一步复习建议。

效果：

- 能检测学生是否真的会分析句子。
- 能避免只靠选择题猜答案。
- 能还原线下精分析作业的核心过程。

### 3.2 五问递进检测

适合仿照豆包聊天的形式。

五问：

1. 句子主干是什么？
2. 语法成分有哪些？分别修饰或说明谁？
3. 考研重难点词汇有哪些？
4. 句子可以怎么切割？
5. 固定搭配或约定俗成表达有哪些？

特点：

- 每次只问一个问题，降低学生压力。
- 前一题答完再进入下一题。
- 全部答完后统一评分。

适用场景：

- 课堂检测。
- 每日一题。
- AI 对话式训练。

### 3.3 七步翻译法作业模式

适合完整训练，不只是检测。

页面结构：

- 抄：原句展示与抄写区。
- 译：结构标注、第一译。
- 查：生词标记与查词记录。
- 改：第二译。
- 对：标准结构和参考译文对照。
- 记：本句记忆清单。
- 滚：加入复习计划。

效果：

- 能直接替代一部分线下精分析笔记。
- 老师可以看到学生每一步是否走过场。

### 3.4 对答案质量检测

对应点评中最常见的问题：学生对答案不细，只看译文，不看结构。

功能：

- 学生提交自查记录。
- 系统要求逐项确认：
  - 主干是否对。
  - 从句类型是否对。
  - 修饰对象是否对。
  - 译文是否按结构。
  - 是否漏掉词组或小词。
- 如果学生只写“差不多”“已订正”，系统判为无效订正。

效果：

- 防止把“对答案”做成抄答案。
- 训练学生知道自己到底哪里不会。

### 3.5 错因标签与薄弱点画像

错误标签建议：

- `no_structure_first`：未先划分结构。
- `main_structure_wrong`：主干错误。
- `predicate_wrong`：谓语识别错误。
- `clause_type_wrong`：从句类型错误。
- `modifier_target_wrong`：修饰对象错误。
- `modifier_level_missing`：一级/二级修饰层级遗漏。
- `what_component_wrong`：what/that/whether/when 等引导词成分错误。
- `post_modifier_vs_adverbial_confused`：后置定语和状语混淆。
- `translation_fragmented`：翻译不成完整句。
- `translation_not_structure_based`：译文没有体现结构。
- `word_meaning_missing`：重点词漏掉。
- `phrase_missing`：固定搭配漏掉。
- `copied_answer`：改或对阶段疑似抄答案。
- `review_not_solid`：滚动复习不牢。

效果：

- 让检测结果能指导下一次训练。
- 老师可以按班级统计共性问题。

### 3.6 少量选择题热身

选择题可以保留，但要少。

适合做选择题的内容：

- 哪一部分是谓语？
- which/that 修饰谁？
- 某个词在本句中是什么意思？
- 这个从句是什么类型？

不适合只做选择题的内容：

- 完整句子主干。
- 句子切分。
- 结构直译。
- 对答案反思。

原则：

- 选择题只占检测总量的 10%-20%。
- 选择题用于激活，不作为主要分数来源。

## 4. 评分体系

综合检测建议 100 分：

| 模块 | 分值 | 评分重点 |
| --- | ---: | --- |
| 主干识别 | 20 | 主语、谓语、宾语/表语/补语是否准确 |
| 语法成分与修饰关系 | 25 | 从句、短语、一级/二级修饰、修饰对象是否准确 |
| 句子切分 | 15 | 是否按结构切分，而不是随意断开 |
| 结构直译 | 20 | 是否完整、通顺、按结构、无关键漏译 |
| 词汇与固定搭配 | 10 | 是否覆盖本句重点词、词性、词组、句型 |
| 自我修正 | 10 | 是否能说清自己哪里错、怎么改 |

评级：

- A：90-100，掌握扎实，能独立分析。
- B：75-89，主干基本正确，但修饰、词组或翻译细节有漏洞。
- C：60-74，能看出部分结构，但分析不稳定，译文常不完整。
- D：0-59，主要依赖感觉，主干和修饰关系不清。

评分方式：

- 早期可以采用 AI 辅助评分 + 标准答案人工抽查。
- 关键班级检测建议保留老师复核入口。
- 系统评分必须给出错误标签，而不是只给分。

## 5. 学生端操作设计

学生端要尽量轻，不要像填复杂表格。

建议页面：

1. 顶部固定显示原句。
2. 下方分步骤填写，每步只有一个核心任务。
3. 每步给简短提示，不展示答案。
4. 支持保存草稿。
5. 提交后才显示标准答案。
6. 错误反馈用颜色和标签，而不是大段吓人的文字。
7. 可以一键加入滚动复习。

交互建议：

- 主干题：三个输入框，主语 / 谓语 / 宾语或表语。
- 语法成分题：结构块标注或短答输入。
- 切分题：在原句中点击插入 `/`。
- 翻译题：大文本框，要求完整句。
- 词汇题：先让学生自己列，再显示系统重点词对照。
- 自我修正：三个短输入框，“我错在 / 原因 / 下次识别方法”。

## 6. 老师端功能

老师需要看到学生是否真的扎实，而不是只看正确率。

功能：

- 班级检测任务发布。
- 指定句子、指定题型、指定步骤。
- 查看学生每一步作答。
- 查看错误标签统计。
- 查看“疑似抄答案”记录。
- 查看学生滚动复习完成情况。
- 按错误标签生成讲评清单。
- 导出学生错因报告。

班级统计示例：

- 主干错误率 35%。
- 从句类型错误率 42%。
- 后置定语/状语混淆 28%。
- 译文不完整 31%。
- 固定搭配漏记 46%。

这类数据比“选择题正确率”更能说明学习是否扎实。

## 7. 现有 JSON 是否够用

结论：现有 JSON 可以作为基础，但不够支撑完整七步翻译法检测。

现有 JSON 已有：

- `original`：原句。
- `translation_literal`：结构直译。
- `chunks`：结构块。
- `role`：结构角色。
- `target_id` 和 `relation`：部分修饰关系。
- `key_words`：重点词。
- `key_phrases`：重点短语。
- `difficulty`：难度。

可以支持：

- 显示原句。
- 主语/谓语/宾语等部分客观检测。
- 重点词和短语检测。
- 部分修饰对象检测。
- 简单结构表展示。

不足：

1. 没有标准主干汇总字段，主干只能从 chunks 推断。
2. 没有标准切分答案。
3. 没有一级修饰、二级修饰的显式层级。
4. `sentence_patterns` 当前为空，不能支撑句型归纳。
5. 没有语法点解释，例如 what 在从句中作什么成分。
6. 没有可接受答案变体，主观题评分困难。
7. 没有常见错误和评分细则。
8. 没有第一译、第二译、自我订正、滚动复习等学习过程数据。
9. 没有题目来源上下文，如段落、定位题、是否为解题阻碍句。
10. 当前只有一篇文章 20 句，覆盖面不够。

### 8.1 示例一：现有 JSON 句子补充后

这个例子来自当前 `kaoyan_2015_english1_text1.json` 中的 `s_003`。原始 JSON 已经有 `original`、`chunks`、`translation_literal`、`key_words`，但如果要做精分析检测，还需要把主干、切分、语法点、评分点和常见错误显式补出来。

```json
{
  "sentence_id": "s_003",
```

## 10. AI 批改要求

AI 批改不能只判断“像不像标准答案”，而要按精分析目标判。

AI 输入应包括：

- 原句。
- 标准主干。
- 标准结构块。
- 标准切分。
- 标准译文。
- 重点词、短语、句型。
- 常见错误标签。
- 学生每一步答案。

AI 输出应包括：

- 总分和分项分。
- A/B/C/D。
- 错误标签。
- 哪些答案可接受。
- 哪些地方必须修改。
- 一条最关键的下次提醒。
- 推荐下一道强化题。

AI 批改原则：

- 主干错是高优先级问题。
- 只写词义、不成句，翻译不得高分。
- 对答案反思空泛，不给自我修正分。
- 漏掉关键小词或固定搭配要扣分。
- 允许译文表达不同，但不允许结构关系错。

## 11. 最小可行版本

第一版不建议一上来做很复杂。

### V1 必做

- 原句显示。
- 五问主观检测。
- 提交后显示标准答案。
- 按规则生成分项反馈。
- 错误标签。
- 加入滚动复习。
- 少量选择题热身。

### V1 数据补充

每句至少补：

- `main_structure`
- `standard_cuts`
- `grammar_points`
- `modifier_levels`
- `translations.literal`
- `rubric`
- `common_errors`

### V2 再做

- 七步翻译法完整作业模式。
- 第一译/第二译对比。
- AI 批改。
- 班级统计。
- 老师复核。
- 专项强化训练。

### V3 再做

- 原文文章级研读。
- 题目定位句与解题关系。
- 语法点跨句归纳。
- 智能滚动复习计划。
- 学生长期能力画像。

## 12. 能否达到实际检测效果

如果只做选择题，不能达到检测精分析是否扎实的效果。学生可以猜，可以凭选项排除，也不需要真的形成完整结构理解。

如果按本文方案做，能明显更接近真实效果：

- 学生必须看原句。
- 学生必须写主干。
- 学生必须说明修饰对象。
- 学生必须切分。
- 学生必须翻译成完整句。
- 学生必须对照并反思。
- 系统能记录错因并安排滚动。

这才是在检测“是否会精分析”，而不是检测“是否会点选项”。
