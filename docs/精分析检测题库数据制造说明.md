# 精分析检测题库数据制造说明

这份文档是给“负责制造检测数据的另一个聊天”使用的。目标不是重做现有文章 JSON，而是在已有 2015 英语一 Text 1 句子库基础上，补充一套独立的“精分析检测题库”。

## 1. 你要读取的已有数据

已有句子数据在：

```text
data/articles/kaoyan_2015_english1_text1.json
```

这个 JSON 已经包含：

- `sentence_id`：句子编号，例如 `s_003`。
- `original`：英文原句。
- `translation_literal`：结构直译。
- `chunks`：已有结构块。
- `key_words`：重点词。
- `key_phrases`：重点短语。
- `difficulty`：难度。
- `year`、`exam_type`、`text_id`：来源信息。

你不要修改这个旧 JSON。它是文章学习页和结构展示页的基础数据。

## 2. 你要制造的新数据

请新建一套独立检测题库，例如：

```text
data/practice/kaoyan_2015_english1_text1_precision_practice.json
```

如果目录不存在，可以新建 `data/practice/`。

新数据通过 `source_sentence_id` 关联旧 JSON 的 `sentence_id`。

关系如下：

```text
文章句子库 data/articles/*.json
        |
        | sentence_id
        v
精分析检测题库 data/practice/*.json
        |
        | practice_id
        v
学生作答记录 attempts/progress
```

本次只制造“题库数据”，不要制造学生作答记录。

## 3. 总体原则

这套检测题库不是普通选择题题库，而是用来检测学生是否真的完成了精分析。

必须覆盖：

1. 是否能先找句子主干。
2. 是否能识别从句、短语、修饰对象。
3. 是否能做结构切分。
4. 是否能按结构翻译成完整中文句。
5. 是否能掌握本句重点词、熟词僻义、固定搭配。
6. 是否能发现自己的典型错误。
7. 是否能进入滚动复习。

选择题可以有，但必须是“诊断陷阱题”，不能是很浅的送分题。

## 4. 第一批建议制造哪些句子

不要一上来做 20 句。第一批建议先做 8-10 句样板，覆盖不同语法点。

优先做这些：

| 句子 | 难度 | 主要检测点 |
| --- | ---: | --- |
| `s_003` | 3 | 一般疑问句外壳、`suggest that` 宾语从句、`suggest` 熟词僻义 |
| `s_004` | 4 | 疑问句、`mean` 宾语、`with` 介词短语附加说明、习语 `writing is on the wall` |
| `s_006` | 4 | `When` 时间状语从句、`as it was...` 比较/省略、并列谓语 |
| `s_007` | 4 | 强调句 `It is ... that ...`、抽象名词、后置修饰 |
| `s_009` | 5 | `unlike` 介词短语、`because` 原因状语从句、`allow sb to do`、长宾语 |
| `s_011` | 5 | 让步/倒装感表达、插入结构、并列主语、抽象词翻译 |
| `s_012` | 5 | `At a time when...`、宾语从句/定语从句、形式主语 `it is bizarre that...` |
| `s_015` | 5 | 定语从句 `who...`、并列结构、`make it difficult to do` |
| `s_016` | 5 | `While` 让步状语从句、强调句、`have most to fear` |
| `s_020` | 5 | `ought to`、`as English history shows`、强调句、插入语、对比结构 |

如果只能先做 5 句，优先做：`s_003`、`s_007`、`s_012`、`s_016`、`s_020`。

每个句子生成一个 practice 对象。建议结构如下：

```json
  "source_sentence_id": "s_003",
  "practice_type": "precision_analysis",
  "version": "v1",
  "source_snapshot": {
    "year": 2015,
    "exam_type": "英语一",
    "text_id": "Text 1",
    "original": "So, does the Spanish crisis suggest that monarchy is seeing its last days?"
  },
  "difficulty": 3,
  "is_key_sentence": true,
  "focus_tags": ["宾语从句", "一般疑问句", "熟词僻义", "主干识别"],
  "why_key": "本句容易把 suggest 误解成“建议”，也容易忽略 that 从句作 suggest 的宾语。",

```

### 5.1 `source_snapshot`

这里要复制旧 JSON 中的关键信息，方便新题库独立阅读。

必须有：

- `year`
- `exam_type`
- `text_id`
- `original`

可以有：

- `translation_literal`
- `difficulty`

注意：`original` 必须和旧 JSON 完全一致，不要改写英文原句。

### 5.2 `focus_tags`

写本句主要训练点。标签要短，便于后续统计。

例子：

```json

```

常用标签建议：

- `主干识别`
- `一般疑问句`
- `宾语从句`
- `定语从句`
- `状语从句`
- `主语从句`
- `形式主语`
- `强调句`
- `倒装`
- `插入语`
- `同位语`
- `后置定语`

## 7. 诊断陷阱选择题设计

每句建议设计 2-4 道选择题，但必须是“诊断陷阱题”。

不要这样问：

```text
suggest 是什么意思？
```

要这样问：

```text
本句中 suggest 最合适的意思是：
A. 建议
B. 表明/暗示
C. 推荐
D. 使想起
```

### 7.1 诊断陷阱题必须包含的字段

```json
{
  "choice_id": "s_003_suggest_meaning",
  "choice_type": "vocab_trap",
  "target": "suggest",
  "error_tag": "familiar_word_unfamiliar_meaning",
  "prompt": "本句中 suggest 最合适的意思是：",
  "options": [
    {
      "label": "A",
      "text": "建议",
      "is_correct": false,
      "trap_reason": "这是 suggest 的常见义，但本句后接 that 从句，语境中是“表明/暗示”。"
    },
    {
      "label": "B",
      "text": "表明/暗示",
      "is_correct": true,
      "trap_reason": null
    },
    {
      "label": "C",
      "text": "推荐",
      "is_correct": false,
      "trap_reason": "与 suggest 的部分语义接近，但本句不是推荐某人或某物。"
    },
    {
      "label": "D",
      "text": "使想起",
      "is_correct": false,
      "trap_reason": "混淆 suggest 和 remind。"
    }
  ],
  "explanation": "So, does the Spanish crisis suggest that... 中，suggest 后接 that 宾语从句，表示“表明/暗示”。"
}
```

### 7.2 有价值的选择题类型

#### 熟词僻义

检测学生是否只背了最常见意思。

例子：

- `suggest`：建议 vs 表明/暗示。
- `case`：情况/案例 vs 论据。
- `claim`：声称 vs 自称。
- `see`：看见 vs 经历/见证/认为。

#### 从句成分

检测学生是否知道从句在主句中作什么。

例子：

```text
that monarchy is seeing its last days 在句中作什么成分？
A. 主语从句
B. 宾语从句
C. 定语从句
D. 状语从句
```

#### 修饰对象

检测学生是否能判断短语或从句修饰谁。

例子：

```text
with their magnificent uniforms and majestic lifestyles 主要补充说明谁？
A. the writing
B. all European royals
C. the Spanish crisis
D. the wall
```

#### 句型识别

检测学生是否能识别强调句、形式主语、倒装等高频结构。

例子：

```text
It is this apparent transcendence of politics that explains... 属于什么结构？
A. 形式主语
B. 强调句
C. 宾语从句
D. 倒装句
```

### 7.3 陷阱选项的要求

每个错误选项都要像学生真的会错的答案，不能乱编。

错误选项应来自：

- 熟词常见义。
- 常见语法误判。
- 修饰对象误判。
- 把形式主语和强调句混淆。
- 把同位语和插入语混淆。
- 把后置定语和状语混淆。
- 只看中文大意造成的误判。

每个错误选项必须写 `trap_reason`。

## 10. 常见错误 `common_errors`

每句至少写 3 个常见错误。常见错误要能和选择题陷阱、AI 批改、老师讲评关联。

```json
"common_errors": [
  {
    "error_tag": "predicate_wrong",
    "description": "只写 suggest，漏掉疑问句中的助动词 does。",
    "diagnosis": "学生没有还原一般疑问句结构。",
    "remediation": "先把疑问句还原成陈述语序：the Spanish crisis does suggest..."
  },
  {
    "error_tag": "clause_type_wrong",
    "description": "没有说明 that 从句作 suggest 的宾语。",
    "diagnosis": "学生看到 that 从句后，没有判断它在主句中的功能。",
    "remediation": "看从句前面的动词，suggest 后面的 that 从句通常说明 suggest 的内容。"
  },
  {
    "error_tag": "familiar_word_unfamiliar_meaning",
    "description": "把 suggest 误译成“建议”。",
    "diagnosis": "学生只记熟词常见义，没有结合语境。",
    "remediation": "遇到 suggest that... 要优先考虑“表明/暗示”。"
  }
]
```

建议统一使用这些错误标签：

- `no_structure_first`
- `main_structure_wrong`
- `subject_wrong`
- `predicate_wrong`
- `object_wrong`
- `clause_type_wrong`
- `modifier_target_wrong`
- `modifier_level_missing`
- `post_modifier_vs_adverbial_confused`
- `emphasis_vs_formal_subject_confused`
- `appositive_vs_parenthesis_confused`
- `familiar_word_unfamiliar_meaning`
- `phrase_missing`
- `translation_fragmented`
- `translation_not_structure_based`
- `copied_answer`
- `self_check_empty`

最终输出文件应该是 JSON 数组：

不要输出 Markdown 表格作为最终题库。可以先用 Markdown 解释，但最终必须给可保存的 JSON。

## 14. 质量检查清单

每做完一个句子，逐项检查：

- `source_sentence_id` 是否能在旧 JSON 中找到。
- `original` 是否和旧 JSON 完全一致。
- 是否有 5-6 个主观任务。
- 是否有 2-4 道诊断陷阱选择题。
- 每个选择题错误选项是否有 `trap_reason`。
- 选择题是否真的对应易错点，而不是送分题。
- 是否有标准主干。
- 是否有标准切分。
- 是否有结构直译和通顺译文。
- 是否有至少 3 个常见错误。
- 常见错误是否能指导学生改进。
- 是否避免把学生作答数据写进题库。

## 15. 不要做的事

- 不要修改 `data/articles/kaoyan_2015_english1_text1.json`。
- 不要只制造普通选择题。
- 不要让错误选项明显离谱。
- 不要只写中文译文而不写结构分析。
- 不要只写“答案略”“见解析”。
- 不要把选择题设计成纯词义题。
- 不要忽略熟词僻义、从句成分、修饰对象、句型混淆。
- 不要把学生作答、分数、复习进度放进题库。
